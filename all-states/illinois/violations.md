# Illinois — Violations / Inspections / Enforcement Research

_Updated 2026-04-18. Covers DCFS "Sunshine" portal per-facility licensing compliance and enforcement data._

## Violations / Inspection Data Source
Illinois uniquely runs a dedicated public-facing violations portal called **DCFS Sunshine**, separate from the agency's main licensing site. Sunshine is the statutory response to the Child Care Act of 1969 transparency mandate.

- **Portal root:** https://sunshine.dcfs.illinois.gov/
- **Provider Lookup (status + basic info):** https://sunshine.dcfs.illinois.gov/Content/Licensing/Daycare/ProviderLookup.aspx
- **Monitoring Reports (violations per facility):** https://sunshine.dcfs.illinois.gov/Content/Licensing/Daycare/MonitoringReports.aspx
- **Investigative Actions (enforcement tracker):** https://sunshine.dcfs.illinois.gov/Content/Licensing/InvestigationActions.aspx
- **Annual Child Care Report:** https://sunshine.dcfs.illinois.gov/Content/Licensing/Daycare/AnnualReports.aspx

The Sunshine portal covers data "gathered on or after January 1, 2010." Pre-2010 records require phone requests at **1-877-746-0829**.

## Data Format
- **ASP.NET WebForms** (note: `.aspx` pages, version string `v 2.3.2.18623` observed). ViewState and `__EVENTVALIDATION` tokens must be preserved across postbacks.
- **Export buttons** (to CSV): confirmed present on both the Provider Lookup page and the Investigation Actions page. The existing `illinois_leads.csv` was in fact generated by driving the Provider Lookup export. The Monitoring Reports page has **no CSV export** — it is interactive look-up only, rendering reports as HTML tables with click-through to per-visit details. Some per-visit reports are served as PDF.
- **Severity coding** used on Monitoring Reports:
  - `CAN` — Child Abuse/Neglect report made
  - `COR` — Violations Corrected
  - `ENR` — Enforcement: Referred for Revocation
  - `MON` — Monitoring activity
  - `NFA` — No findings / no violations

## Freshness
- Portal self-reports "48 to 72 hour" propagation after a licensing action is entered by DCFS staff.
- No stated update schedule — data is written real-time as inspections are closed out.
- Annual Child Care Report (statewide aggregate) posted annually, typically Q2 for prior calendar year.

## Key Fields
### Per-facility Monitoring Report (HTML / PDF)
- Provider Name, License Number, Facility Type
- Visit Date
- Visit Type (Annual / Permit / Complaint / Monitoring / Post-Violation Follow-up)
- Violation Code (per Rule 407 / 406 / 408)
- Violation Narrative
- Corrective Action Plan
- Status (Corrected / Open / Repeat)

### Investigation Actions page columns
- Provider Name, License, County
- Action Code (CAN / COR / ENR / MON / NFA)
- Status (Substantiated / Unsubstantiated / Pending)
- Date
- Repeat Violation flag

## Scraping / Access Strategy
### Provider Lookup (existing path — confirmed working)
```
# Start session
curl -c cookies.txt 'https://sunshine.dcfs.illinois.gov/Content/Licensing/Daycare/ProviderLookup.aspx'
# POST: Export button — preserve __VIEWSTATE and __EVENTVALIDATION from prior GET
curl -b cookies.txt -c cookies.txt -X POST \
     --data '__VIEWSTATE=...&__EVENTTARGET=btnExport&...' \
     'https://sunshine.dcfs.illinois.gov/Content/Licensing/Daycare/ProviderLookup.aspx' \
     -o 'Daycare Providers.csv'
```
This is what produced the 8,586-row `illinois_leads.csv`.

### Investigation Actions export (enforcement lead list)
Same ASP.NET postback pattern against `InvestigationActions.aspx` → exports a CSV of every Substantiated/Pending enforcement action with the provider name. This is the richest source of hot leads for ComplianceKit — every facility with a recent Substantiated or Pending violation is an intent signal.

### Monitoring Reports (per-facility deep detail)
1. No bulk export. Iterate over each License Number from the Provider Lookup CSV.
2. Each Monitoring Reports query requires a License Number or facility lookup.
3. Parse the result HTML table; follow `ReportID` links to individual reports (some PDF).
4. Rate-limit aggressively — Sunshine is lightly provisioned and 2 req/sec has tripped temporary IP blocks in prior automation.

### Hot-leads query
- Filter Investigation Actions CSV where `Action IN ('CAN','ENR')` and `Date >= now - 90 days` → high-intent compliance prospects.
- Cross-reference with Provider Lookup CSV by License Number to enrich with phone + address.

## Known Datasets / Public Records
- **DCFS Sunshine:** the authoritative dataset.
- **data.illinois.gov:** no dedicated child-care violations dataset as of 2026-04 (verified). DCFS publishes Sunshine instead.
- **Illinois Early Childhood Asset Map (IECAM):** https://iecam.illinois.edu/ — aggregate enrollment and capacity data, not violations.
- **Chicago Tribune "Broken by Design" / BGA** investigations have previously published PDF datasets derived from DCFS Sunshine; no current ongoing data release.
- **Winnie "How to Look up Daycare Violations":** https://winnie.com/resources/how-to-look-up-daycare-violations — references Sunshine as the canonical IL source.

## FOIA / Open-records Path
- **Statute:** Illinois FOIA, 5 ILCS 140.
- **DCFS FOIA:** https://dcfs.illinois.gov/about-us/policy-rules-and-forms/foia.html — request via `DCFS.FOIA@illinois.gov`. Response in 5 business days (extendable to 10).
- **Useful for:** pre-2010 violation records, un-redacted complaint investigation files, staff-of-concern records (CANTS inquiries), internal Plan of Correction documents, and a full bulk export of post-2010 monitoring reports if scraping is infeasible. DCFS has granted CSV dumps of monitoring-report detail on request.
